## 🧠 第三章：核心概念解释（新手必读）

### 3.1 什么是 batch_size 和 labeled_bs？

```
batch_size = 96    ← 每个 step 总共处理的 2D 切片数
labeled_bs  = 24   ← 其中有标签的切片数
                   ← 无标签切片数 = 96 - 24 = 72
```

- `labeled_bs` 控制有标签数据的利用率
- 一般 `labeled_bs = 全体有标签切片数 / 每 epoch 期望步数`
- ACDC 标准：7 个病人 = 136 张有标签切片，`labelnum=7` 时 `labeled_bs=24` 约 5~6 step 覆盖一轮

### 3.2 什么是 DDP（分布式数据并行）？

```
单 GPU 模式：               4 GPU DDP 模式：
┌─────────────┐            ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐
│   GPU 0     │            │GPU 0 │ │GPU 1 │ │GPU 2 │ │GPU 3 │
│  batch=24   │            │ bs=24│ │ bs=24│ │ bs=24│ │ bs=24│
│  显存使用~XXG│            │ 各负责各自 batch，梯度汇总
└─────────────┘            └──────┘ └──────┘ └──────┘ └──────┘
                              全局 batch = 24×4 = 96
```

**DDP 的好处**：
- 4 倍数据吞吐量
- 更大的全局 batch → BN 统计量更准
- 每个 GPU 上的 batch_size 不变（不增加单卡显存负担）

**启动方式**：必须用 `torchrun --nproc_per_node=N` 启动，不能直接用 `python`

### 3.3 什么是 AMP（自动混合精度）？

```
传统 FP32 训练：             AMP 训练：
每个数值 32 位              大部分计算 16 位
                          权重 32 位（保持精度）
                          梯度自动缩放（GradScaler）
                          
显存节省 ~30-40%
速度提升 ~1.5-2×
精度无损
```

**谁可以用 AMP**：任何支持 CUDA >= 11.0 的 GPU 都可以。NVIDIA L20（Ada Lovelace）对 FP16/BF16 有硬件加速。

### 3.4 训练的两个阶段

```
预训练（pre_train）                 自训练（self_train）
┌──────────────────────┐           ┌──────────────────────────┐
│ 只用有标签数据        │           │ 有标签 + 无标签数据       │
│ 标准监督训练          │   加载    │ EMA 教师生成伪标签        │
│ 学习基础分割能力      │ ← 权重 →  │ BCP 双向 copy-paste      │
│ pre_iterations 步    │           │ CMC 互补掩码一致性       │
└──────────────────────┘           │ max_iterations 步（含pre）│
                                   └──────────────────────────┘
```

预训练权重保存到 `model/BCP/ACDC_{exp}_{labelnum}_labeled/pre_train/unet_best_model.pth`
自训练从这里加载，然后继续训练。

## 🔄 第四章：用旧权重继续训练

### 4.1 场景说明

你有之前在 `code/` 下跑出来的权重（来自原始单 GPU 脚本），想在 `05.30/code/` 的新脚本中继续训练。

### 4.2 新旧脚本差异分析

| 维度 | 旧脚本 (`code/BCP_CMC_fusion.py`) | 新脚本 (`05.30/code/BCP_CMC_fusion_ddp.py`) |
|------|----------------------------------|--------------------------------------------|
| GPU | 1 卡 | 1~4 卡均可 |
| 精度 | FP32 | FP32 或 AMP |
| state_dict | 无 `module.` 前缀 | 有 `module.` 前缀（DDP 包装后） |
| EMA teacher | 从 student 权重复制初始化 | 从 student 权重复制初始化 |
| 预训练流程 | 自动串行 | 新增 `--load_path` 跳过 |

### 4.3 方法一：用 `--load_path`（推荐，最清晰）

无需修改代码，脚本已经支持：

```bash
# 4 GPU + AMP 恢复
torchrun --nproc_per_node=4 BCP_CMC_fusion_ddp.py \
    --ddp --amp \
    --batch_size 96 --labeled_bs 24 \
    --load_path /root/my-bcp-experiments/code/model/BCP/ACDC_BCP_CMC_fusion_7_labeled/self_train/unet_best_model.pth

# 1 GPU + AMP 恢复
python BCP_CMC_fusion_ddp.py \
    --gpu 0 --amp \
    --batch_size 24 --labeled_bs 12 \
    --load_path /path/to/unet_best_model.pth
```

当指定了 `--load_path` 时，脚本会自动跳过 pre-train 阶段，直接进入 self-train 阶段，并从指定路径加载权重。

### 4.4 方法二：`--pre_iterations=0` + 复制权重（最简单，不碰代码）

```bash
# Step 1：把旧权重复制到预训练路径
mkdir -p ./model/BCP/ACDC_BCP_CMC_fusion_ddp_7_labeled/pre_train
cp /path/to/unet_best_model.pth \
   ./model/BCP/ACDC_BCP_CMC_fusion_ddp_7_labeled/pre_train/unet_best_model.pth

# Step 2：设置 --pre_iterations=0 跳过预训练
torchrun --nproc_per_node=4 BCP_CMC_fusion_ddp.py \
    --ddp --amp \
    --batch_size 96 --labeled_bs 24 \
    --pre_iterations 0 \
    --max_iterations 30000
```

脚本运行流程会变成：
1. pre_train(args) 被调用，但 `pre_iterations=0` → 循环直接不执行 → 只创建目录
2. self_train() 在 `pre_snapshot_path` 中找到你复制的权重 → 加载 → 继续训练

### 4.5 注意事项（重要）

| 注意事项 | 说明 |
|---------|------|
| ✅ `module.` 前缀自动处理 | 旧权重无 `module.`，新脚本 DDP 后有 `module.`，[`load_net()`](05.30/code/utils/train_utils.py:190) 会自动适配 |
| ✅ EMA teacher 初始化 | 旧权重只保存了 student，加载时 student 和 EMA teacher 都会从同一个权重初始化，EMA 在前几步预测稍弱，但很快赶上 |
| ✅ 优化器状态 | 每卡 batch=24 不变（原版就是 24），momentum/weight_decay 兼容 |
| ⚠️ max_iterations | 恢复后建议设为**总迭代数**（不是剩余数），因为 `consistency_rampup` 从 0 开始 |
| ⚠️ 学习率 | `warmup_poly_lr` 会从 `base_lr * 0.1` 开始 warmup，首次会略低 |
| ❌ 不要同时用两种方法 | `--load_path` 和 `--pre_iterations=0` 都可以，选一个就行 |

#### 📂 方案 B：你的数据是 3D volume（一个文件就是一个完整 3D 扫描）

医学影像常见格式：一个 `.nii.gz` 或 `.h5` 文件包含整个 3D 扫描。

```python
class BaseDataSets(Dataset):
    def __init__(self, base_dir=None, split='train', num=None, transform=None):
        self._base_dir = base_dir
        self.transform = transform
        self.sample_list = []
        
        if split == 'train':
            with open(self._base_dir + '/train.list', 'r') as f:
                patient_list = f.readlines()
            patient_list = [p.replace('\n', '') for p in patient_list]
            
            # ===== 关键：把每个病人的每张切片展开为一条记录 =====
            for patient in patient_list:
                h5f = h5py.File(f"{base_dir}/{patient}/mri_norm2.h5", 'r')
                num_slices = h5f['image'].shape[0]
                h5f.close()
                for slice_idx in range(num_slices):
                    self.sample_list.append((patient, slice_idx))
        # ...
    
    def __getitem__(self, idx):
        patient, slice_idx = self.sample_list[idx]
        h5f = h5py.File(f"{self._base_dir}/{patient}/mri_norm2.h5", 'r')
        image = h5f['image'][slice_idx]     # [H, W]
        label = h5f['label'][slice_idx]     # [H, W]
        h5f.close()
        
        sample = {'image': image, 'label': label}
        if self.transform:
            sample = self.transform(sample)
        return sample
```

> 参考官方 LA 数据集的 [`LAHeart`](05.30/code/dataloaders/dataset.py:93) 类的写法（第 93-128 行），它对 3D 数据按 volume 展开。

这个函数的作用：告诉训练循环"N 个病人 = 多少张 2D 切片"，从而确定有标签/无标签集合的边界。

**修改方案**：

**方案 A（推荐）**：直接计算

```python
def patients_to_slices(dataset, patiens_num):
    # 如果你的数据平均每个病人约 20 张切片：
    return patiens_num * 20
```

**方案 B**：硬编码（适用于你确定每个病人的切片数）

```python
def patients_to_slices(dataset, patiens_num):
    if "YourDataset" in dataset:
        ref_dict = {"1": 15, "2": 30, "5": 75, "10": 150}
    else:
        return patiens_num * 20  # 兜底
    return ref_dict[str(patiens_num)]
```

> ⚠️ 如果 `patiens_num` 不在字典中会报 `KeyError`，所以兜底很重要。

## 🔧 第六章：常见问题排查

### Q1：OOM（显存不足）

```
RuntimeError: CUDA out of memory.
```

**解决方法**（按优先级排列）：

1. **检查 batch_size 是否正确**：`batch_size ÷ GPU数` 不应超过单卡显存上限
   - L20 (46GB)：单卡上限约 40~50（`patch_size=256×256` 时）
   - 4×L20：`batch_size=96` → 每卡 24 → ✅ 安全
   - 4×L20：`batch_size=192` → 每卡 48 → ❌ 可能 OOM
2. **开启 AMP**：`--amp` 省 30% 显存
3. **减少 num_workers**：`create_ddp_dataloader(..., num_workers=2)` 默认 4，可降为 2
4. **检查其他进程**：`nvidia-smi` 看是否有其他程序占用显存

### Q2：`torchrun` 找不到

```
bash: torchrun: command not found
```

```bash
# 方法 1：用 python -m torch.distributed.run
python -m torch.distributed.run --nproc_per_node=4 BCP_CMC_fusion_ddp.py --amp --ddp

# 方法 2：确认 torch 版本 ≥ 2.0
python -c "import torch; print(torch.__version__)"

# 方法 3：用 run_exp.sh（自动处理路径）
bash ../run_exp.sh -e BCP_CMC_fusion_ddp -g 0,1,2,3 -n 4 --torchrun ...
```

### Q3：`RuntimeError: DistBackend` 或 `Address already in use`

```
RuntimeError: Address already in use
```

DDP 默认用了 29500 端口，已经被占用了：

```bash
# 指定其他端口
torchrun --master_port=29501 --nproc_per_node=4 BCP_CMC_fusion_ddp.py ...
```

### Q4：验证时卡死 / 无输出

DDP 模式下，只有 rank 0 做验证（日志只从 rank 0 输出）。如果等了很久没输出：

1. 确保 `is_main_process()` 有正确的 rank 判断
2. 检查 [`test_single_volume_all_metrics`](05.30/code/utils/metric_utils.py:37) 中是否有 `.cuda()` 硬编码（如有，需改为 `.to(device)`）
3. 在 DDP 验证前确认调用了 `torch.distributed.barrier()`

### Q5：模型加载报 `missing keys` 或 `unexpected keys`

```
Missing key(s) in state_dict: "module.encoder.conv1.weight"
Unexpected key(s) in state_dict: "encoder.conv1.weight"
```

这是 **`module.` 前缀不匹配**问题。

**原因**：旧权重（无 DDP 包装）的 state_dict 中 key 以 `encoder.conv1.weight` 开头；新脚本中模型被 DDP 包装后，key 以 `module.encoder.conv1.weight` 开头。

**自救方案**：

```python
# 加载时手动处理
checkpoint = torch.load('old_weights.pth')
new_state_dict = {}
for k, v in checkpoint.items():
    if not k.startswith('module.'):
        new_state_dict['module.' + k] = v  # 加上 module. 前缀
    else:
        new_state_dict[k] = v

model.load_state_dict(new_state_dict)
```

**自动方案**：本项目 [`load_net()`](05.30/code/utils/train_utils.py:190) 已经自动处理了这种情况，你不需要手动做任何处理。

| 参数 | ACDC | LA |
